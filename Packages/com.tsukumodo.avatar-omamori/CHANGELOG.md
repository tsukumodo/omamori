## [0.8.0] - 2026-07-24
### 追加
- [Shader] Emission（意図しない発光）チェックを追加。「肌にテクスチャを入れたら光り輝くアバターになった」型の事故を、ビルド前に警告として検出する。対象は lilToon（Main）と Unity Standard / Standard (Specular setup)。発光がオンでも色が黒い・極端に暗いマテリアルは「実際には光らない」ものとして検出しない
- 上記チェックに「発光をオフ」の自動修正を追加。発光色は残したままトグルだけをオフにするので、あとから同じ色で光らせ直せる。Undo（Ctrl+Z）で元に戻せる。確認ダイアログで「他の箇所にも反映されること」「メニューで光らせる設計の場合は壊れうること」を明示する
- ロック済みマテリアル（Poiyomi/ThryEditor でロックしたもの）は、設定値が実状態を反映せず直せないため検出対象から除外。対応外のシェーダーも同様に、プロパティ名を推測せず検出しない

### 変更

### 修正
- 「カードを保存」を押していないのに、スクリプトの再コンパイル直後に「カード画像の書き出しに失敗しました」というエラーダイアログが出ることがある問題を修正（v0.7.0 から存在）。おまもりのウィンドウをチェック結果が表示された状態で開いたまま、他のツールの導入などで Unity がスクリプトを再コンパイルすると発生していた

## [0.7.0] - 2026-07-13
### 追加
- チェック結果をSNS共有向けのカード画像（PNG・1200×630）として書き出す機能を追加。結果サマリー右の「カードを保存」ボタンから保存先を選ぶと、エラー/警告/情報の件数・自動修正件数・日付（年月日のみ）・ツールバージョンをまとめた抹茶トーンのカードを出力する（全クリア時は金色の「✓」付きで「問題は見つかりませんでした」を表示）。アバター名・パスなど個人を特定しうる情報はカードに載せない

## [0.6.0] - 2026-06-08
### 追加
- 自動修正の Before/After 表示用フィールドを `CheckResult` に追加（`ValueLabel` / `BeforeValue` / `AfterValue`）。Animator Layer Weight / Missing Script / VRC Avatar Descriptor 重複の3項目で、修正前後の値（予告）を保持
- 結果カードに自動修正の Before/After（予告）を1行で表示（例: `FX Layer / Weight: 0 → 1`）。「修正」ボタンを押すと値がどう変わるかを事前に予告し、「直す力」を可視化
- セッション内の自動修正履歴を「修正履歴」Foldout で表示。各修正の直前/直後の実値（予告値ではなく実測）と修正対象を時刻順に保持（上限100件、新しい順表示、Ping ボタン・クリアボタン付き、EditorWindow 終了で消える）
- 利用統計のローカル記録機構を追加。チェック・修正の種別ごとの回数／チェック実行回数／日付（年月日のみ）／ツールバージョンを、プロジェクト内の `Library/com.tsukumodo.avatar-omamori/usage-stats.json` に記録（自動送信なし・ローカル完結）。アバター名・パス・シーン名・PC名・ユーザー名・時分秒などの個人を特定しうる情報は記録しない
- 「Tools > つくも堂 > 使用統計を見る」ウィンドウを追加。蓄積データの表示・「フィードバックとしてコピー」（クリップボードへコピー＋つくも堂 X の DM 導線）・統計のクリア・収集の無効化/再開ができる
- 初回起動時に、利用統計をローカル記録する旨をメインウィンドウ内に1行で告知（ダイアログは出さない）。収集の有効/無効はプロジェクト単位で永続化

## [0.5.0] - 2026-04-28
### 追加
- アバタールート指定時にチェックを自動実行するように変更（従来は「チェック実行」ボタン押下が必須。ボタンは引き続き手動再チェック用に利用可能）
- 自動修正の基盤を追加（CheckResult に FixAction を持たせ、UI に「修正」ボタンを表示）
- #8 Animator Layer Weight=0 の自動修正を実装（Weight=0 を 1 に変更、Undo 対応）
- #4 Missing Script の自動修正を実装（GameObject 上の Missing Script を一括削除。Unity の API 仕様で Undo 不可だが、Missing Script は元々参照が壊れているため実質的なデータ損失はなく、事前確認でその旨を明示）
- #1 VRC Avatar Descriptor 重複の自動修正を実装（複数の Descriptor から「どれを残すか」を選ぶドロップダウンで解消。本体が推奨、選んだもの以外は一括削除で Undo 対応）
- 共通基盤に SkipConfirm フラグを追加（修正処理が独自の UI（ダイアログ・ドロップダウン等）を出す場合に、共通基盤側の事前確認と自動再チェックをスキップする）

## [0.4.0] - 2026-04-11
### Added
- [SDK] Animator Layer Weight チェック（FXレイヤーの Weight=0 を検出・警告）
- [SDK] 同期パラメータ上限チェック（同期パラメータの合計ビット数が256bitを超える場合をエラー）

## [0.3.1] - 2026-04-03

- [MA] MAObjectToggleSelfRefCheck を MAObjectToggleCheck に改名（ObjectToggle全般のバリデーションとして整理）

## [0.3.0] - 2026-04-03

### Added
- [MA] 装飾物の未セットアップ検出チェック（Armatureを持つ子オブジェクトにMA Merge Armature / Bone Proxyが未設定の場合を警告。ターゲット未設定も検出）

- [SDK] Expression Menu設定済み・Parameters未設定の検出
- [MA] ObjectToggle 空ターゲット検出（トグルリストのターゲットが空の場合を警告）

## [0.2.0] - 2026-03-28

### Added
- [Unity] Missing Script 検出チェック（アバター配下のMissing Scriptを検出・致命的エラー）
- [Shader] シェーダー未検出（ピンクマテリアル）チェック（シェーダーが見つからないマテリアルを検出・警告）

## [0.1.0] - 2026-03-27

### Added
- VRC Avatar Descriptor 重複チェック
- MA MenuItem 未接続チェック（祖先に MenuInstaller がない MenuItem を検出）
- MA ObjectToggle 自己参照チェック（自身の GameObject をトグル対象にしているケースを検出）
- EditorWindow UI（Tools > アバター改変おまもり）
- Severity 別（Error / Warning / Info）のグループ表示
- オブジェクト選択（Ping + Selection）機能
